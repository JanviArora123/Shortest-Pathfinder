About

Captain! Are you excited to begin your next space adventure? Pull up your socks and get ready for your journey on Mars.We are aware that when you're out there, you'll need a trustworthy
path finder to aid you. And thus, we put together this application for you!

The Grid: A graph, with each cell representing a node. Each cell is connected to adjacent cells horizontally, vertically, and diagonally by invisible edges that (by default) cost to traverse.
Click on Start Node and End Node buttons to move around the Source and Destination nodes respectively.

Weights/Walls: Drag or click anywhere on the grid to place or remove weights or walls.

Weighted cells: These cells have edges that cost more to traverse horizontally or vertically. They represent environments that are difficult to cross.
(Note : Breadth First Search is an unweighted algorithm.)
Walled cells: These cells do not have edges connecting them to adjacent cells. They represent obstacles.

Speed Selection: Use the dropdown to toggle between simulation speeds for two node path-finders.

Station Node: Select the Allow Stations checkbox, click on the Station Node button and click anywhere to add multiple stations. Feel free to experiment and don't forget to have fun!

Problem Statements And Algorithms:
1. Find the shortest path from the source to the destination. You can define the state of the space using various features, choose an algorithm from the panel, set the visualization speed,
select weights of all the weighted nodes, and click visualize to see the pathfinder build the path. Algorithms : AStar, ID Depth First Search, Breadth First Search, IDAStar, Best First Search,
Dijkstra.
2. Traveling Salesman - Define intermediate stations and find the least cost path that starts from the source, covers all intermediate points (stations), reaches the destination at some
point and returns to the source. The Traveling Salesman algorithm does this for you.
3. Path optimization with a maximum cost constraint - Suppose your rover is on a mission. Every intermediate station is a place for you to stock up on food supplies. But alas, your rover
has a limited amount of fuel, and you must reach the destination before you run out. The Multiple Stops algorithm tries to optimize at every point, so that with the
available fuel, you can reach your destination, while visiting as many stations as you can, on your way there.
